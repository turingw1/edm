# DG-TWFD EDM Teacher-Proxy Target Ablation

This folder runs a no-training simulation experiment on top of the official EDM
pretrained checkpoints and online sampler. The experiment asks:

> Since DG-TWFD evaluates a model as a transition operator `M(t,s,x_t)`, which
> prediction target is best aligned with compositional reuse?

Only the target space changes. The EDM checkpoint, online reference rollout,
time sampler, 4-step generation schedule, FID evaluator, seeds, and target-space
approximation rule are fixed.

The script also reports an `edm` row from the unmodified EDM `generate.py`
4-step sampler. This is a sanity baseline for checking whether the checkpoint,
FID reference, generated-image directory, and evaluator are wired correctly.

## Targets

For each source state `x_t` and cleaner target time `s`, EDM online Heun rollout
provides the reference transition `x_s* = Phi_EDM(t -> s, x_t)`. We then apply
the same restricted approximation rule to one of three target spaces:

```text
endpoint: z = x_s*
residual: z = x_s* - x_t
velocity: z = (x_t - x_s*) / (t - s)
```

The induced transition map is reconstructed as:

```text
endpoint: M(t,s,x_t) = A(z)
residual: M(t,s,x_t) = x_t + A(z)
velocity: M(t,s,x_t) = x_t - (t - s) A(z)
```

`A` is shared across targets and defaults to fixed-range uniform quantization.
Optional block averaging can be enabled in the config, but the default keeps
`block_size=1` because spatial block averaging destroys CIFAR-10 sample quality
and is too severe for the FID@4 proxy. This is a capacity-limited teacher proxy,
not a trained student.

## Metrics

Match MSE:

```text
E_match = E || M(t,s,x_t) - x_s* ||_mse^2
```

Normalized semigroup defect:

```text
Delta =
  || M(t,r,x_t) - M(s,r,M(t,s,x_t)) ||_mse^2
  / (eps_Delta + || x_r* - x_t ||_mse^2)
```

FID@4 is computed from samples generated by recursively applying the induced
map on the same 4-step EDM sigma schedule.

## Preparation

For CIFAR-10, no raw training dataset is required. The experiment samples from
the public EDM checkpoint and computes FID against the public CIFAR-10 reference
statistics. The first run needs network access to download:

```text
EDM CIFAR-10 checkpoint:
https://nvlabs-fi-cdn.nvidia.com/edm/pretrained/edm-cifar10-32x32-cond-vp.pkl

CIFAR-10 FID reference:
https://nvlabs-fi-cdn.nvidia.com/edm/fid-refs/cifar10-32x32.npz

Inception detector used by fid.py:
https://api.ngc.nvidia.com/v2/models/nvidia/research/stylegan3/versions/1/files/metrics/inception-2015-12-05.pkl
```

Recommended server setup:

```bash
cd /data2/yl7622/Zhengwei/DG-TWFD/refs/edm
conda activate dg
export PYTHONPATH="$PWD:${PYTHONPATH:-}"
export DNNLIB_CACHE_DIR=/data2/yl7622/Zhengwei/DG-TWFD/.cache/dnnlib
mkdir -p "$DNNLIB_CACHE_DIR"
```

`dnnlib` is a local module inside the EDM repository. If `import dnnlib` fails,
you are either not running from `refs/edm` or `PYTHONPATH` does not include the
EDM root. Check the environment before prefetching:

```bash
python - <<'PY'
import dnnlib
import torch
import PIL
import scipy
print("dnnlib:", dnnlib.__file__)
print("torch:", torch.__version__)
print("cuda:", torch.cuda.is_available())
PY
```

If any non-`dnnlib` import fails, install the EDM dependencies in the active
environment:

```bash
python -m pip install click tqdm pillow scipy requests psutil
python -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

Optional prefetch command:

```bash
python - <<'PY'
import dnnlib

urls = [
    "https://nvlabs-fi-cdn.nvidia.com/edm/pretrained/edm-cifar10-32x32-cond-vp.pkl",
    "https://nvlabs-fi-cdn.nvidia.com/edm/fid-refs/cifar10-32x32.npz",
    "https://api.ngc.nvidia.com/v2/models/nvidia/research/stylegan3/versions/1/files/metrics/inception-2015-12-05.pkl",
]
for url in urls:
    print("prefetch", url)
    with dnnlib.util.open_url(url):
        pass
PY
```

If the server has no internet access, download these files elsewhere, copy them
to the server, and edit the config fields:

```json
"checkpoint": "/path/to/edm-cifar10-32x32-cond-vp.pkl",
"fid_ref": "/path/to/cifar10-32x32.npz"
```

For FID without internet, also make sure the Inception detector has been cached
under `$DNNLIB_CACHE_DIR/downloads`, or run one online prefetch once on a
machine that shares the same cache directory.

## CIFAR-10

Smoke run:

```bash
python experiments/dg_twfd_teacher_proxy/scripts/run_target_ablation.py \
  --config experiments/dg_twfd_teacher_proxy/configs/DG_TWFD_cifar10_target_ablation.json \
  --num-samples 128 \
  --eval-num-triplets 128 \
  --batch 512 \
  --eval-batch 256 \
  --fid-batch 512
```

Larger run:

```bash
python experiments/dg_twfd_teacher_proxy/scripts/run_target_ablation.py \
  --config experiments/dg_twfd_teacher_proxy/configs/DG_TWFD_cifar10_target_ablation.json \
  --num-samples 50000 \
  --eval-num-triplets 10000
```

## Summaries

The run script writes summaries automatically. To regenerate summaries from an
existing run:

```bash
python experiments/dg_twfd_teacher_proxy/scripts/summarize_results.py \
  --run-dir experiments/dg_twfd_teacher_proxy/outputs/DG_TWFD_cifar10_target_ablation
```

Final paper-facing files are:

```text
experiments/dg_twfd_teacher_proxy/results/summary_cifar10.md
experiments/dg_twfd_teacher_proxy/results/summary_cifar10.csv
experiments/dg_twfd_teacher_proxy/results/summary_imagenet64.md
experiments/dg_twfd_teacher_proxy/results/summary_imagenet64.csv
```

Raw samples, FID logs, and per-target metrics are under:

```text
experiments/dg_twfd_teacher_proxy/outputs/DG_TWFD_<dataset>_target_ablation/
```

The CSV fields are:

```text
dataset,target,fid4,defect,match_mse,checkpoint,sampler_name,num_samples,seed
```

The `edm` row has only `FID@4`; `defect` and `match_mse` are blank because it is
the unmodified sampler baseline rather than a target-space induced map.

These CSV files are intended to feed the final paper table directly.
